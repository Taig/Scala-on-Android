Similar to {{ names.name( 'java' ) }}'s {{ command( '.class' ) }} file, {{ command( '.dex' ) }} contains the compiled bytecode of your app. But instead of creating one {{ command( '.class' ) }} file for each class (like javac does it), the {{ command( '.dex' ) }} file contains the entire byte code of your app (including dependencies). But you are discouraged to use the {{ names.name( 'multidex' ) }} feature to avoid stripping out the {{ names.name( 'scala' ) }} overhead because its usage does not come without issues. {{ names.name( 'multidex' ) }} is intended for large codebases.
