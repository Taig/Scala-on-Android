1. project-root/
    1. project/
      1. Build.scala

        Your actual build configuration. Here you specify your (sub-) projects, manage application dependencies and configure ProGuard.

        You can also manage your build via {{ command( '/build.sbt' ) }}. However, {{ command( '/project/Build.scala' ) }} is written in actual Scala code, is slightly more expressive and allows you to keep all of your configuration files in the {{ command( '/project' ) }} folder.

      2. build.properties

        An optional, but recommended file that specifies the sbt version for this build.

      3. plugins.sbt

        Dependencies for sbt are managed in this file. This is necessary, as we need to rely on Android SDK Plugin for SBT here.

    2. src/
      1. main/
        1. AndroidManifest.xml
        2. res/

          Common Android resource folders such as {{ command( 'drawable/' ) }} and {{ command( 'values/' ) }} go here.

        3. java/

          Java source files that may interact with the Scala sources and vice versa.

        4. scala/

          The actual application code is located in this folder if you're not building a Java app with sbt (which is also possible).

      2. test/
        1. AndroidManifest.xml
        2. res/
        3. java/
        4. scala/
      3. androidTest/

        A folder introduced by the Android plugin to host instrumentation tests.
