The {{ command( 'TR' ) }} mappings are generated on every compile, when the {{ names.name( 'sbt-plugin' ) }} detects changes in the {{ command( 'R' ) }} class. Your {{ names.abbreviation( 'ide' ) }} usually helps you to keep the {{ command( 'R' ) }} up to date behind the scenes, but it won't do that for the {{ command( 'TR' ) }} class. You can still achieve a similar effect if you open your project {{ names.abbreviation( 'sbt' ) }} shell and run {{ command( '~compile' ) }}. The tilde before the command tells {{ names.abbreviation( 'sbt' ) }} to repeat this task whenever it detects a file change.

{{ command( 'TR' ) }} mappings only work for {{ names.abbreviation( 'xml' ) }} widgets that use the {{ command( 'android:id="@+id/xxx"', 'xml' ) }} property. If you chose not to use the {{ command( 'TR' ) }} class you may want to disable it for performance reasons in your project's {{ names.abbreviation( 'sbt' ) }} configuration via {{ command( 'typedResources := false', 'scala' ) }}.
