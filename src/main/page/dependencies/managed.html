Managed dependencies

The preferred way to add dependencies are managed dependencies. They offer fine-grained control about the dependencies, yet they are still easy to maintain. Library dependencies are maintained via the {{ names.abbr( 'sbt' ) }} configuration key {{ command( 'libraryDependencies' ) }}. The {{ names.name( 'play' ) }} documentation provides a good {{ sourcelink( 'play-framework-documentation-sbt-dependencies', 'introduction' ) }} to managed, unmanaged dependencies and resolves.

Besides simple {{ names.name( 'java' ) }} or {{ names.name( 'scala' ) }} libraries, you can also reference {{ names.name( 'android' ) }} aar or apklib packages. Usually, the {{ names.name( 'sbt-plugin' ) }} is able to detect and handle these dependencies correctly, but in some cases it might be necessary to mark them exeplicitly with {{ command( 'aar()' ) }} or {{ command( 'apklib()' ) }} (e.g. {{ command( 'aar( "com.android.support" % "appcompat-v7" % "22.2.0" )' ) }}).

After adding a managed dependency, you have to run {{ command( 'sbt reload' ) }} and you should also run {{ command( 'sbt clean' ) }} to enforce a rebuild of the {{ names.name( 'proguard' ) }} cache.
