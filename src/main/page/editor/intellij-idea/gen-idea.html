gen-idea

It is now necessary to create the {{ names.abbreviation( 'ide' ) }} configuration files via the gen-idea {{ names.abbreviation( 'sbt' ) }} plugin. This step will hopefully soon be superfluous, but for now it is still obligatory. {{ names.name( 'intellij-idea' ) }} should be able to simply import an {{ names.name( 'sbt-plugin' ) }} based plugin, but in its current state it fails to do so.

gen-idea is a {{ names.abbreviation( 'sbt' ) }} task that generates configuration files for the {{ names.name( 'intellij-idea' ) }}. To install the {{ sourcelink( 'sbt-plugin-gen-idea', 'gen-idea' ) }} plugin create the file {{ command( '~/.sbt/0.13/plugins/gen-idea.sbt' ) }} (the filename is arbitrary) with the contents below.

{{ code( 'page/editor/intellij-idea/gen-idea.scala.html' ) }}

This makes the gen-idea {{ names.abbreviation( 'sbt' ) }} command globally available.

Now, navigate your terminal to the project root directory and run {{ command( 'sbt gen-idea' ) }}. This will create {{ names.name( 'intellij-idea' ) }} 13 configuration files, but newer versions of {{ names.name( 'intellij-idea' ) }} are able to convert the configuration to their appropriate format.

You can now chose the Open option of the {{ names.name( 'intellij-idea' ) }} and select the root directory of your project.

Converting the project files only needs to be done once at the initial setup. When the {{ names.abbreviation( 'ide' ) }} finished loading you may delete the {{ command( '/projectFilesBackup' ) }} folder.
