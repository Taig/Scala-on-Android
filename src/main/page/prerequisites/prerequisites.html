• Java Development Kit (JDK)

    Android supports Java 7, so the JDK 7 should also be your choice for now. If you already have a newer version of the JDK installed, this will work as well, but then you have to specify your compile target properly to version 1.7, which will be explained in the build tools topic.

    {% call alert( warning ) %}

    With the upcoming release of Scala 2.12, Java 8 will be required. Therefore Scala 2.12 can not be used for Android development until Google introduces Java 8 support for Android.

    {% endcall %}

  • Android Software Development Kit (SDK)

    The Android SDK is a comprehensive tooling suite. It comes with applications that compile Java to Android bytecode, debugging and profiling applications, the Android emulator and a whole lot more.

    Download and Install the SDK. Then open your terminal and run {{ command( 'android sdk' ) }}. The Android SDK Manager window will pop up and ask you to install Android version specific documentation, development APIs, sources and disk images for emulation. For a quick start it is sufficient to stick with the default selection.

    {{ image( '/asset/image/android-sdk-manager-settings.png', 'A screenshot of the Android SDK manager with default settings' ) }}

  • Build Tool

    Build tool integration plugins for Scala on Android are currently available for sbt, Gradle and Maven. sbt, the Scala Build Tool, is the Scala standard build tool and its configuration files are also written in Scala. The Android SDK Plugin for SBT is a very mature and well maintained plugin, furthermore most Scala IDEs integrate sbt nicely. Therefore sbt is the build tool of choice for this documentation.

    {% call alert( warning ) %}

    If you do not have any experiences with Scala on Android and other build tools than sbt, it is highly recommended to chose sbt, since many parts of this documentation are focused on its configuration details!

    {% endcall %}

    A comparison of available build tools and detailed instructions for sbt is waiting for you in the next chapter.

  • Editor / IDE

    You are more or less free to use any editor of your choice. With IDEs the integration process might cause a little trouble and depends on your build tool of choice. Please refer to the Editor / IDE chapter for further information. The recommended tool is IntelliJ IDEA, which is also used as reference throughout the documentation.
