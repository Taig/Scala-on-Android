If you already have a newer version of the {{ names.abbreviation( 'jdk' ) }} installed, this will work as well, but then you have to specify your compile target properly to version 1.7, which will be explained in the build tools topic.

  • {{ names.render( 'android-sdk' ) }}

    The {{ names.abbreviation( 'android-sdk' ) }} is a comprehensive tooling suite. It comes with applications that compile {{ names.name( 'java' ) }} to {{ names.name( 'android' ) }} bytecode, debugging and profiling applications, the {{ names.name( 'android' ) }} emulator and a {{ sourcelink( 'android-tools', 'whole lot more' ) }}.

    {{ sourcelink( 'android-download', 'Download' ) }} and {{ sourcelink( 'android-install', 'install' ) }} the {{ names.abbreviation( 'sdk' ) }}. Then open your terminal and run {{ command( 'android sdk' ) }}. The {{ names.abbreviation( 'android-sdk-manager' ) }} window will pop up and ask you to install {{ names.name( 'android' ) }} version specific documentation, development {{ names.abbreviation( 'api' ) }}s, sources and disk images for emulation. For a quick start it is sufficient to stick with the default selection.

  • Build Tool

    Build tool integration plugins for {{ names.name( 'scala-android' ) }} are currently available for {{ sourcelink( 'sbt-plugin', names.abbreviation( 'sbt' ) ) }}, {{ sourcelink( 'gradle-plugin', names.name( 'gradle' ) ) }} and {{ sourcelink( 'maven-plugin', names.name( 'maven' ) ) }}. The {{ names.render( 'sbt' ) }}, is the {{ names.name( 'scala' ) }} standard build tool and its configuration files are also written in {{ names.name( 'scala' ) }}. The {{ names.name( 'sbt-plugin' ) }} is a very mature and well maintained plugin, furthermore most {{ names.name( 'scala' ) }} {{ names.abbreviation( 'ide' ) }}s integrate {{ names.abbreviation( 'scala' ) }} nicely. Therefore {{ names.abbreviation( 'sbt' ) }} is the build tool of choice for this documentation.

    {% call alert( warning ) %}

    If you do not have any experiences with {{ names.name( 'scala-android' ) }} and other build tools than {{ names.abbreviation( 'sbt' ) }}, it is highly recommended to chose {{ names.abbreviation( 'sbt' ) }}, since many parts of this documentation are focused on its configuration details!

    {% endcall %}

  • Editor / {{ names.abbreviation( 'ide' ) }}

    You are more or less free to use any editor of your choice. With {{ names.abbreviation( 'ide' ) }}s the integration process might cause a little trouble and depends on your build tool of choice. Please refer to the Editor / {{ names.abbreviation( 'ide' ) }} chapter for further information. The recommended tool is {{ names.name( 'intellij-idea' ) }}, which is also used as reference throughout the documentation.
